Decision Log

Date Decision Rationale
Feb 14, 2026 Calendar inherits account-scoped and team/admin settings from Mail Calendar and Mail share the same account and org, so account-scoped settings that Mail has (Profile, Add Accounts, Password & Security, Theme) and org-scoped team settings (Members, Invite, Roles) logically apply to Calendar too. Other org-scoped settings (Billing, Security) and account-scoped settings Mail doesn't surface (Connected Accounts, Notifications) stay as dashes.
Feb 14, 2026 Single-product tiers show no checks The tier header already identifies which product owns those settings (e.g., "Mail only"). Showing a check under the obvious product column is redundant â€” the grouping alone communicates ownership.
Feb 14, 2026 Go column removed Go is a new product with TBD settings. Including it with question marks or dashes adds noise without information. Will re-add once Go's settings surface is defined.
Feb 14, 2026 Standalone HTML over React component The audit table is a reference artifact, not a product feature. A zero-dependency HTML file is cheaper to produce, easier to share, and doesn't add build complexity to the prototype.
